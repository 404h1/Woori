# 통신사·빅테크 보이스피싱 방지 솔루션 현황 (2026.05)

> 우리은행 SSAFY AI 금융소비자보호 아이디어 경진대회 사전조사
> 작성일: 2026-05-20
> 범위: 한국 통신 3사(SKT·KT·LGU+) + 글로벌 빅테크(삼성·Google·Apple) + 해외 사례(O2 UK)

한국은 2024~2025년에 걸쳐 "통화 단(端)에서 실시간 AI가 보이스피싱을 잡아낸다"는 영역을 **세계 최초로 상용화**한 국가이다. 본 문서는 각 사업자의 공식 발표·뉴스룸 자료를 기반으로 ① 구현 방식, ② 기능, ③ 서비스 포인트, ④ 은행 연동, ⑤ 법적 근거, ⑥ 한계를 정리하고, 마지막에 **"은행이 비집고 들어갈 빈틈"**을 도출한다.

## 1. SKT 에이닷(A.) — 스캠뱅가드(ScamVanguard)

- 공식 사이트: <https://adot.ai/>
- 뉴스룸(보이스피싱): <https://news.sktelecom.com/211089>
- 뉴스룸(A.X 4.0 오픈소스): <https://news.sktelecom.com/213534>
- 뉴스룸(보안 신기능): <https://news.sktelecom.com/215392>
- A.X 4.0 오픈소스: <https://github.com/SKT-AI/A.X-4.0>

### 1.1 서비스 개요
- **서비스명**: 에이닷 전화(A. 전화) + AI 보이스피싱 탐지(스캠뱅가드, ScamVanguard)
- **상용화/주요 발표일**: 2025년 4월 8일 (IBK기업은행 및 에이닷 정식 적용)
- **A.X(에이닷 엑스) 4.0 오픈소스 공개**: 2025년, Hugging Face에 표준/경량 2종 배포
- **2025년 9월**: 에이닷 검색에 A.X 4.0 + GPT-5 도입, 'AI 메시지' 보안 기능 추가

### 1.2 구현 방식
- **모델 구성**: 스캠뱅가드는 4종 AI 모듈로 구성
  1. AI 미끼문자(스미싱) 탐지 시스템
  2. 피싱 시도 채팅 탐지 시스템
  3. **보이스피싱 통화 패턴 분석 AI** (통화 흐름·키워드 패턴 학습)
  4. 본인확인 분석 AI (타인 명의 인증 시도 탐지)
- **기반 LLM**: A.X 4.0 (Qwen2.5 기반에 대규모 한국어 데이터 추가 학습한 한국어 특화 LLM). 로컬 환경 운영 가능, 데이터 보안 고려 설계
- **데이터 흐름**: 통신 정보(SKT 망 데이터) + 금융 데이터(은행 거래) 연계 → 서버 측 이상탐지 통합 서비스. 개인정보보호위원회 사전검토 통과

### 1.3 주요 기능
- 보이스피싱 의심 번호 수신 시 통화 경고 메시지 표시
- AI 의심 번호 자동 차단('AI 안심차단')
- 문자 보안 기능 'AI 메시지': 사칭·사기·링크주의 라벨 부착
- 이용자 평가 정보 공유(크라우드소싱형 평판)

### 1.4 서비스 포인트 (고객 실질 가치)
"전화를 받기 전부터 받은 후, 그리고 송금하기 전까지" 통신-금융 양단에서 한 번 더 걸러주는 안전망. 특히 IBK 사례처럼 **은행 거래 직전 차단**까지 이어진다는 점이 차별점.

### 1.5 사용자 수 / 효과
- **에이닷 월간 성과**: 약 19만 건의 보이스피싱 의심 번호 통화에 대한 경고 메시지 제공(월간)
- **IBK기업은행 2주 사전 테스트**: 26건 피해 예방, 약 5억 9천만 원 손실 방지

### 1.6 은행 연동
- **IBK기업은행 SurPASS 패키지**에 스캠뱅가드 탑재 — 통신사가 탐지한 이상 신호를 은행이 받아 **이체·출금 선제 차단**에 활용
- 국내 최초로 **통신 정보 + 금융 데이터 연계** 이상탐지 통합 서비스 표방

### 1.7 법적 근거 / 한계
- 개인정보보호위 사전검토 통과
- 한계: 통화 내용 자체보다는 **번호 평판·통화 패턴 메타 분석** 중심으로 보임(통신비밀보호법 회피). 통화 내용 실시간 STT 분석의 깊이는 LGU+·KT 대비 약함

## 2. KT 후후(whowho / whox2) — AI 보이스피싱 탐지 서비스 2.0

- 공식 사이트: <https://whox2.com/>
- 실시간 AI 보이스피싱 탐지 안내: <https://whox2.com/kt/vfd>
- 안내 페이지(상세): <https://image.whox2.com/whox2appweb/html/voicePhishing_lms.html>

### 2.1 서비스 개요
- **서비스명**: 'AI 보이스피싱 탐지 서비스 2.0' (후후 앱 내 탑재)
- **출시일**: 2025년 7월 29일 (2.0, 국내 최초 화자인식 + 딥보이스 통합 상용화)
- **1.0**: 2025년 상반기, KT 자체 데이터 기반 문맥 분석 시작 (탐지율 99% 인터뷰)

### 2.2 구현 방식
- **온디바이스(On-Device) 분석**: 통화 내용이 단말 내부에서만 분석되어 외부 서버로 전송되지 않음 → 개인정보 유출 차단
- **3단 분석**:
  1. **문맥 분석 AI** — 보이스피싱 범죄 시나리오 학습, 통화 내용 의미 분석
  2. **화자 인식(성문 매칭)** — 알려진 보이스피싱 범죄자 음성과 유사도 비교 (국내 최초 상용화)
  3. **딥보이스 탐지** — TTS·AI 합성 음성 식별. 음성의 미세한 '파형 왜곡', '메타데이터 불일치' 등을 AI가 정밀 분석
- **학습 데이터**: KT 자체 보이스피싱 신고 통화 데이터(전국 신고·후후 사용자 신고 기반)

### 2.3 주요 기능
- 실시간 통화 분석 → 의심 시 **단계별 팝업/진동 경고**
- 보이스피싱 사고 음성과 **유사도가 높을 경우** 추가 경고
- 기계/AI 변조 음성(딥보이스)일 경우 추가 경고
- 통신사 무관 — 후후(whowho) 앱 설치 시 사용 가능

### 2.4 서비스 포인트 (고객 실질 가치)
"내가 듣고 있는 이 목소리가 **실제 사람인가, 합성인가, 이미 범죄에 쓰였던 목소리인가**"를 단말이 알려준다. 가족·지인 사칭형 딥보이스에 대한 1차 방어선.

### 2.5 사용자 수 / 효과
- 2025년 상반기 누적 **1,460만 건의 통화 데이터** 실시간 분석
- 탐지 정확도: **91.6%** (2025년 상반기) → **97.2%** (2025년 4분기)
- 피해 예방 추정액: 약 **710억 원**(2025 상반기 누적)
- 지원 단말: 삼성 갤럭시 S23 시리즈 이상 (iOS는 개발 중)

### 2.6 은행 연동
- 현재 공식 발표상 직접적 은행 FDS 연동은 SKT·LGU+ 대비 제한적
- 정부 주도 'AI 기반 보이스피싱 통신서비스 공동대응 플랫폼'(2026~2027 구축 예정)에 참여 예상

### 2.7 법적 근거 / 한계
- **ICT 규제 샌드박스 승인** — 성문(목소리 지문)은 민감 개인정보(통신비밀보호법·개인정보보호법 대상)이므로 화자 인식 학습·운용을 위해 규제 샌드박스로 합법화
- 한계: 갤럭시 일부 기종에만 동작, iOS 미지원. 성문 DB는 KT 신고 데이터 위주

## 3. LG U+ 익시오(ixi-O) — 안티딥보이스 + KB FDS 연동

- 공식 사이트(B2B 페이지): <https://www.lguplus.com/biz/contact/ixi>
- 안티딥보이스 보도자료: <https://www.lg.co.kr/media/release/29093>
- 단말 지원 확대: <https://www.lg.co.kr/media/release/29042>
- 익시오 체험단 모집: <https://news.lguplus.com/20653>
- 익시오 2.0 출시: <https://www.mt.co.kr/tech/2025/10/20/2025102008525184343>
- KB국민은행 연동 MWC26 발표: <https://www.newsis.com/view/NISX20260223_0003522421>

### 3.1 서비스 개요
- **서비스명**: 익시오(ixi-O) — AI 통화 앱
- **1.0 출시**: 2024년 11월 (실시간 보이스피싱 탐지)
- **안티딥보이스 탑재**: **2025년 6월 30일** (세계 최초 온디바이스 안티딥보이스 상용화)
- **2.0 출시**: 2025년 10월 (개인 맞춤형 AI 강화, 지난 통화 검색)
- **KB국민은행 FDS 실시간 연동**: 2026년 2월 MWC26에서 공개

### 3.2 구현 방식
- **온디바이스 AI** — 고객 정보·통화 데이터가 스마트폰에만 저장, 통신사 서버 비전송 (세계 최초 온디바이스 안티딥보이스)
- **기반 모델**:
  - 거대 LLM: LG AI연구원 **엑사원(EXAONE)**
  - 자체 sLLM: **익시젠(ixi-GEN)** — 온디바이스 추론용
- **학습 데이터**: LG유플러스 자체 통신 데이터 **약 3,000시간 / 약 200만 건** 통화 학습
- **안티딥보이스 알고리즘**: 위조 목소리의 부자연스러운 발음, 음성 주파수의 비정상 패턴 탐지. **약 5초** 통화 데이터로 위변조 판단

### 3.3 주요 기능
- 실시간 보이스피싱 탐지
- **안티딥보이스** (위변조 음성 판별)
- AI 대신 전화받기(AI 콜 응답)
- 통화 녹음·요약, 영상 통화
- 지난 통화 의미 검색(2.0)
- 안티딥페이크(합성 영상 탐지) — 개발 중
- 통신사 무관(통신3사 누구나) — 갤럭시 S21 이상 지원

### 3.4 서비스 포인트 (고객 실질 가치)
"내 스마트폰 안에서, 5초 안에 가짜 목소리를 잡아낸다." **온디바이스 = 프라이버시 보장 + 즉시성**. AI가 대신 받아주는 기능까지 결합되어 '통화 보안 비서' 포지션.

### 3.5 사용자 수 / 적용 범위
- 체험단 5,000명 모집 후 정식 서비스로 확장(통신사 무관 개방)
- 2026년 중 신규 AI 기능 추가 예정

### 3.6 은행 연동 (**가장 진보된 사례**)
**KB국민은행 + LG U+, MWC26에서 'AI 기반 금융-통신 보이스피싱 실시간 대응 협력 체계' 공개 (2026.02)**

데이터 흐름:
1. 익시오가 단말에서 ① 의심 음성 패턴, ② 악성 URL 접근, ③ 악성 앱 설치 시도를 탐지
2. 탐지 즉시 KB국민은행 FDS(이상거래탐지시스템)로 데이터 공유
3. KB국민은행이 해당 계좌에 **즉시 결제(이체) 정지** 또는 정밀 모니터링 전환
4. 종전 "피해자 신고 후 사후 대응" → "**범죄 실행 순간 선제 차단**"으로 패러다임 전환

### 3.7 법적 근거 / 한계
- 온디바이스 처리로 통신비밀보호법·개인정보보호법 회피 설계
- 한계: 갤럭시 S21 이상 안드로이드만 지원(아이폰 미지원), 5초 분석 지연, 통화가 아닌 **타 채널(메신저, 영상)**은 별도 솔루션 필요

## 4. 삼성 갤럭시 AI — '보이스피싱 의심 전화알림'

- 공식 사이트: <https://www.samsung.com/sec/galaxy-ai/>
- 보도자료: <https://news.samsung.com/kr/삼성전자-갤럭시-스마트폰-ai로-보이스피싱-스팸-한>

### 4.1 서비스 개요
- **서비스명**: 갤럭시 기본 전화 앱의 '보이스피싱 의심 전화알림'
- **최초 도입**: 2024년 7월 (갤럭시 Z폴드7·Z플립7)
- **확대**: One UI 8.0 이상 탑재 모든 갤럭시
- **2025년 8월** 보안 기능 대폭 추가 발표

### 4.2 구현 방식
- **온디바이스 AI** — 통화 내용이 단말 내부에서만 분석, 외부 서버 비전송
- 의심 키워드·대화 패턴·음성 위변조 여부 종합 분석
- 한국 사용자 대상으로 KT(후후)·SKT·LGU+ 각사 AI 솔루션과 **이중으로** 동작 (단말사 + 통신사 레이어 분리)

### 4.3 주요 기능
- 모르는 번호 통화 내용을 분석해 **2단계 알림**:
  1. "의심"(보이스피싱 의심)
  2. "경고"(보이스피싱 감지)
- 스팸 통합 차단

### 4.4 서비스 포인트
"통신사를 가리지 않고 단말 차원에서 기본 제공" — 통신사 앱을 깔지 않아도 동작하는 **기본값 안전망**.

### 4.5 한계
- 갤럭시 단말 한정(One UI 8.0+)
- KT/LGU+처럼 성문·딥보이스 정밀 탐지까지는 아직 미흡
- 은행과의 직접 연동은 없음 — 알림에 그침

## 5. Google Scam Detection — Gemini Nano 온디바이스

- 공식 블로그: <https://blog.google/technology/safety-security/how-were-using-ai-to-combat-the-latest-scams/>
- Chrome Enhanced Protection: <https://9to5google.com/2025/05/08/chrome-enhanced-protection-gemini-nano/>

### 5.1 서비스 개요
- **서비스명**: Scam Detection in Google Messages & Phone by Google
- **출시**: 2024년 말 시범 → 2025년 미국 본격 확대
- **Chrome Enhanced Protection** + Gemini Nano: 2025년 5월

### 5.2 구현 방식
- **온디바이스 Gemini Nano** — 통화·메시지 내용을 단말 내부에서 분석, 데이터 클라우드 전송 없음
- 대화 패턴 학습 — '처음엔 무해하지만 점차 사기로 발전'하는 **conversational scam** 탐지에 특화

### 5.3 주요 기능
- 통화 중 실시간 사기 의심 알림
- 모르는 발신자와 **화면 공유 중 금융앱 실행 시 경고** (Android in-call scam protection)
- Chrome: 웹페이지 실시간 분석 → 사기 페이지 20배 더 많이 적발

### 5.4 서비스 포인트

## 9. 핵심 시사점 — 통신사가 못 하는 빈틈 (= 은행이 비집고 들어갈 영역)

통신 3사 + 삼성/Google이 **통화·메시지 단(端)에서 의심을 잡아내는 영역은 사실상 포화 상태**다. 하지만 다음 7가지 영역에서 통신사·단말사는 본질적으로 한계가 있고, 이것이 **은행이 직접 들어가야 할 빈틈**이다.

### 빈틈 ① 통화 후 ~ 송금 직전 '20분 공백'
- 통신사 탐지는 통화 중·통화 직후에서 끝난다. 피해자가 통화를 끊고 **앱·창구·ATM으로 이동하는 5~30분**은 무방비.
- LG U+×KB가 익시오→KB FDS 실시간 연동으로 일부 해결했지만, 이는 **그 통신사·그 은행 가입자**에 한정.
- **은행의 자리**: 모든 통신사·단말 무관, **은행 앱·창구·ATM에서 송금 직전 1회 더 묻는 AI 게이트** — "지금 통화하다 오셨나요? 누가 시켰나요?" 자체 STT/대화형 검증.

### 빈틈 ② 비통화 채널 (메신저·SNS·대면)
- 통신사 솔루션은 **통화·SMS 채널**에 묶여 있다.
- 카카오톡·텔레그램·인스타그램 DM·**대면 사기(검사 사칭 직접 만남)**·QR 결제 사기는 통신사 레이더 밖.
- **은행의 자리**: 송금 행위 자체의 맥락(수취인 신규 여부, 평소 시간대 이탈, 이체 직전 화면 캡처 의심 등)을 단서로 비통화 사기까지 커버.

### 빈틈 ③ 단말 분절 — 아이폰 사용자·구형폰 사용자
- 익시오(S21+), 후후 2.0(S23+), 삼성 기본 기능(One UI 8+), Google Scam Detection(Pixel/일부) — **모두 안드로이드 신형 단말 중심**.
- **iPhone 사용자, 65세 이상 구형폰 사용자, 폴더폰 사용자**는 통신사 AI 보호를 거의 못 받는다 (Apple Call Screening은 보이스피싱 특화 아님).
- 보이스피싱 **주피해층이 바로 이 노인·구형폰 사용자**라는 점에서 매우 큰 빈틈.
- **은행의 자리**: 단말 의존 없는 **은행 앱·창구·콜센터** 채널에서의 보호. 특히 노인 대상 창구 직원 AI 보조(Whisper류 STT + 이상 발언 감지).

### 빈틈 ④ 다중 은행·다중 계좌 시나리오
- 통신사가 단일 은행(IBK·KB)과 1:1 연동해도, 범죄자는 **여러 은행에 분산 송금**하는 패턴.
- 통신사는 어느 은행 계좌인지 모른다 — 단지 "위험 신호"만 던질 뿐.
- **은행의 자리**: 금융결제원/오픈뱅킹 데이터를 활용해 **수취인 계좌의 직전 입출금 패턴**(머니뮬, 단기간 다수 입금) 분석. 통신사가 절대 못 보는 시야.

### 빈틈 ⑤ '정상적으로 보이는' 송금 — 본인의 자발적 송금
- 검사·자녀 사칭형은 피해자가 **본인 의지로 송금**하므로 단순 FDS·통신사 탐지를 우회한다.
- LG U+×KB도 '의심 음성/악성URL'이라는 통신 신호가 있어야 작동.
- **은행의 자리**: 행동 신호 — **이체 직전 검색 기록(은행 앱 내), 이체 화면 체류 시간, 평소와 다른 입력 패턴, 송금 메모 자연어 분석** ("국세 환급", "안전계좌" 같은 키워드).

### 빈틈 ⑥ 사후 회수 / 골든타임 30분
- 통신사는 사기 탐지까지만, **돈이 빠져나간 후의 환수**는 은행만 가능.
- **은행의 자리**: 의심 거래 자동 지급정지 → **수취 은행 간 실시간 핫라인** → 골든타임 30분 내 자금 동결 자동화. 현재 수동·전화·팩스 기반.

### 빈틈 ⑦ 법적 책임·보상
- 통신사는 탐지 실패해도 책임 없음. **금융사고 보상은 결국 은행의 영역**(전기통신금융사기특별법).
- **은행의 자리**: 보상 리스크를 안는 주체로서 **자체 검증 레이어**를 갖추는 것이 필수. "통신사가 못 잡았으니 우리도 책임 없다"는 통하지 않는다.

## 10. 우리은행 관점 — 차별화 전략 후보

위 빈틈을 종합하면 우리은행이 가져갈 수 있는 포지셔닝은 다음 3가지이다.

1. **"송금 직전 마지막 1km"** — 단말·통신사 무관, 우리WON뱅킹 앱과 영업점·ATM에서 작동하는 **이체 직전 AI 인터뷰**(STT + LLM 대화형 검증). 통신사 신호 + 행동 신호 + 수취인 계좌 신호 3중 결합.
2. **"노인·아이폰 사용자 전용 안전 채널"** — 통신사 AI 솔루션의 단말 사각지대를 정조준. 콜센터 상담사 AI 보조 + 영업점 직원 단말의 보이스피싱 의심 신호 표시.
3. **"수취 은행 간 골든타임 30분 자동화"** — 우리은행이 주도하는 은행 공동 의심거래 실시간 동결 플랫폼. 통신사 탐지 데이터(SKT/KT/LGU+ 모두)를 표준 API로 수집해 **타행 송금까지 동결** 가능하게.
