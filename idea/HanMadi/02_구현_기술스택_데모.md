## 2. 기술 스택 + 왜 이 기술이어야 하는가

### 2-1. 온디바이스 LLM — **Llama 3.2 3B (INT4 quantized)**

**왜?**
- **시니어 보호 = 통신 끊겨도 작동**해야 함 (사기범이 통화 중 데이터 차단 가능)
- **프라이버시** — AI 기본법(2026.1.22) 데이터 주권 의무 충족
- **3~8B는 CES 2026 표준** — 일상 처리 충분
- **선례**: LG U+ ixi-O 이미 상용, 갤럭시 AI 기본 탑재

**대안 비교:**
| 모델 | 크기 | 한국어 | 비용 | 채택 |
|---|---|---|---|---|
| Llama 3.2 3B | 3B | 좋음 (fine-tune) | 무료 | ✅ |
| Qwen 2.5 7B | 7B | 매우 좋음 | 무료 | 백업 |
| GPT-4o (서버) | — | 최상 | 비용 | 보조 (서버 RAG 한정) |
| 솔라(Solar) 미니 10.7B | 10.7B | 한국어 최적 | 무료 | M9 업그레이드 후보 |

### 2-2. 음성 인식 STT — **OpenAI Whisper-large-v3 + 한국어 fine-tune**

**왜?**
- **한국어 발화 인식률 95%+** (시니어 사투리·발음 부정확도 포함)
- 신한 자체 LLM "발화인식 95%"와 동급
- 사투리·노인 발음 robustness 검증

**Wake-word는 Picovoice Porcupine** — 1MB 미만, 99.5% 정확도, 사용자 등록 가능 ("도와줘", "이상해", "사기야")

### 2-3. SECURE 영상통화 — **WebRTC + SRTP + DTLS**

**왜?**
- **통신망 음성 사칭 우회** = 가족 클로닝 차단의 결정타
- WebRTC는 글로벌 표준, Chrome/Safari/Android 네이티브 지원
- E2E 암호화로 우리은행조차 통화 내용 못 들음 → 개인정보 우려 해소

**보완: 딥페이크 탐지**
- Microsoft Azure / 자체 모델 — 영상통화 중 실시간 얼굴·음성 인공합성 신호 탐지
- 가족 영상통화가 딥페이크면 즉시 경고

### 2-4. 시나리오 분류 LLM — **우리은행 운영GPT(RAG) 확장**

**왜?**
- 이미 우리은행 자산 — 직원용 RAG가 고객용으로 확장
- 사기 시나리오 DB는 차세대 FDS '시나리오 자가생성'과 공유 자산
- ASAP 플랫폼 패턴 실시간 RAG

**입력:** STT 텍스트 + 발신자 번호 + 최근 거래 + 위치
**출력:** {보이스피싱 / 인지혼란 / 사기진행 / 단순도움} + 권고액션 + 신뢰도

### 2-5. 거래 LOCK 엔진 — **우리은행 FDS + 신규 ResponseEngine**

**왜?**
- 차세대 FDS(2026 상반기 오픈)와 자연 통합
- 5분 lock → AI 분류 결과로 자동 연장(12시간) 또는 자동 해제

### 2-6. 가속도계 — **안드로이드 SensorManager / iOS CoreMotion**

**왜?**
- 시각장애·말 못함 케이스의 트리거
- 3축 가속도 패턴 인식 (3회 흔들기 within 2초)
- 오발동 차단: 운동·이동 중 자동 비활성화

### 2-7. 그림카드 — **유니버설디자인(UD) + 발달장애 친화**

**왜?**
- 한국발달장애인협회·서울대 UD 가이드 협업
- 글자 X, 픽토그램 4장: 🆘 도움 / ⚠️ 사기 같음 / ❓ 혼란 / 🚨 긴급
- Mayer Multimedia Principle (이미지 > 텍스트)

### 2-8. ASAP 플랫폼 연계 API — **금융보안원**

**왜?**
- 사기 발신번호 자동 등록 → 산업 차원 보호 ↑
- 통합대응단 자동 신고 → 신고 누락률 60%+ → 5% 미만 기대

## 3. 데이터 흐름 — 발동 시 (3초 시퀀스)

### T+0초: 트리거
```
사용자: "도와줘!" (또는 SOS 버튼 / 3-shake / 그림카드)
   ↓
Porcupine Wake-word: 0.1초 매칭
   ↓
WON뱅킹 SDK Dispatch
```

### T+0.5초: 즉시 LOCK
```
WON뱅킹 백엔드 호출:
{
  "action": "instant_lock",
  "user_id": "encrypted_user_id",
  "duration_seconds": 300,
  "scope": "all_outbound_transfers"
}
→ 모든 송금·청약 즉시 차단
```

### T+1초: 가족 발신
```
가족 그룹 조회 → 1순위 가족 푸시
WebRTC 영상통화 채널 개설
가족 폰 알림: "엄마가 SOS 발동, 영상통화 받으세요"
```

### T+1.5초: AI 위시퍼 음성
```
TTS 출력: "어떤 상황이세요? 5분간 송금은 안전하게 멈췄어요. 가족에게 영상통화 발신 중입니다."
동시에 마이크 ON → Whisper STT 시작
```

### T+3초: 시나리오 분류
```
운영GPT(RAG) 호출:
입력: {
  "user_speech": "검찰이라고 했는데, 4천만원 보내라고...",
  "caller_number": "070-XXXX-XXXX",
  "recent_transactions": [...],
  "location": "서울 강북구"
}
출력: {
  "scenario": "voice_phishing_high",
  "confidence": 0.94,
  "recommended_action": [
    "lock_extend_12h",
    "auto_report_asap",
    "block_caller_number",
    "notify_family_priority_1"
  ]
}
```

### T+5초: 후속 액션
```
1. 거래 LOCK 12시간 연장
2. 통합대응단 ASAP API 자동 신고 (POST /api/v1/report)
3. 발신번호 ASAP 사기 번호 DB 등록
4. 가족에게 "보이스피싱 의심 시나리오" 음성 브리프
5. 영업점(시니어플러스) 콜백 일정 자동 등록
```

## 4. 핵심 코드 인터페이스 (PT용 의사코드)

```python
# 1) 트리거 감지
class HanMadiTrigger:
    def __init__(self):
        self.wake_word = Porcupine(keywords=["도와줘", "이상해", "사기야"])
        self.shake_detector = ShakeDetector(threshold=3, window_sec=2)
        self.sos_button = SOSButton(always_visible=True)
        self.picture_card = PictureCard(cards=4)

    def on_trigger(self, source: str):
        # source ∈ {"voice", "button", "shake", "picture"}
        WONBankAPI.instant_lock(duration_sec=300)
        FamilyChannel.start_video_call(priority=1)
        AIWhisper.start_listening(language="ko")
        return TriggerEvent(time=now(), source=source)

# 2) 시나리오 분류 (서버 RAG)
def classify_scenario(stt_text, context):
    prompt = build_rag_prompt(
        user_input=stt_text,
        recent_transactions=context.transactions,
        caller_id=context.caller,
        location=context.location,
        knowledge_base=["우리은행 사기 DB", "ASAP 패턴", "금감원 주의보 2026"]
    )
    result = WooriOperatingGPT.classify(prompt)
    # result = {scenario, confidence, recommended_actions}
    return result

# 3) 후속 액션 디스패치
def dispatch_actions(scenario, user):
    actions = SCENARIO_PLAYBOOK[scenario]
    for action in actions:
        if action == "lock_extend_12h":
            WONBankAPI.lock_extend(user, hours=12)
        elif action == "auto_report_asap":
            ASAPClient.report(user.encrypted_id, scenario)
        elif action == "block_caller_number":
            CallerBlockList.add(user.recent_call.number)
        elif action == "notify_family":
            FamilyChannel.brief(user.family_priority_1, scenario)
```

## 6. 보안·프라이버시 답변 (Q&A 대비)

| Q | A |
|---|---|
| 음성 항상 듣고 있나? | Wake-word(1MB Porcupine)만 매칭. **STT는 발동 후에만** 켜짐. 비교: 시리·빅스비와 동일 방식 |
| 외부 서버 전송? | Wake-word·시나리오 분류는 **온디바이스 LLM** 처리. 영상통화는 E2E SRTP. 우리은행도 통화 내용 못 봄. |
| 통신비밀보호법? | 사용자 옵트인 + 가족 사전 동의. AI 기본법 데이터 주권 의무 충족. |
| 사기범 우회? | 4중 트리거 + 자동 lock + ASAP 산업 차단. 단일 우회 불가. |
| 오발동 누적 시? | LOCK 5분 기본 (사용자 본인 해제 가능). 12시간은 가족 확인 후만. ML 학습으로 정밀도 ↑. |
| 시니어가 등록 어려움? | 시니어플러스 영업점에서 5분 설명. WON뱅킹 자동 권유 (65세+). |
