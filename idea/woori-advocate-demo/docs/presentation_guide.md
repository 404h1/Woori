### 슬라이드 12 — ROI: 3축 수익 모델  (≈45초)

**화면 구성:**
- 3개 큰 카드:
  - **(A) 회피:** "DLF급 1회 회피 = 197억 ~ 1.4조"
  - **(B) 감경:** "실태평가 우수 → 과징금 최대 75% 감경 = 평균 37.5억/건"
  - **(C) 라이센스 (Phase 5):** "5개국 53개 은행 × $1M + customer fee = 1,000~3,000억/년"
- Break-even 라벨: "P1 6개월 종료 시 (DLF 1건 회피로 충당)"

## 3. Q&A 30개 사전 답변

### 3-1. Top 5 — 반드시 외울 것

#### Q1. "은행이 자기 상품을 안 팔리게 방해하는 셈인데, 지점 KPI 타격은?"
> 고위험 상품 1건 무리 유치 이익보다, 불완전판매 민원으로 인한 지점 감점·배상 청구 리스크가 정량적으로 약 20배 큽니다. Advocate 는 부적합 고객을 사전에 걸러 지점 최대 감점 요인을 원천 방어합니다. 행동경제학적으로도 'Healthy Friction' 은 적합성 원칙 충족의 필수 과정입니다.

#### Q2. "고객이 Advocate 보고도 가입해서 손실 나면 책임은?"
> Advocate 는 직원 책임을 갈음하지 않습니다. 다만 고객이 5가지 리스크를 항목별로 체크하고 서명한 기록이 남기에, 분쟁 시 '은행이 형식 서류를 넘어 AI 변호인까지 두어 인지시켰음' 을 입증하는 결정적 **Safe Harbor** 가 됩니다.

#### Q3. "병원비 결제 내역까지 활용 = 개인정보 위반?"
> 새로운 수집 없습니다. 우리은행 앱에 이미 적재된 자사 거래 내역만 **온프레미스** 우리GPT 로 연산. 이용약관 내 '금융상품 적합성 분석 목적' 활용 동의가 이미 포함되어 있고, Advocate 호출 시 '본 분석에 거래 이력이 사용됩니다' 고지 팝업도 제공됩니다.

#### Q4. "환각으로 멀쩡한 가입을 막으면?"
> 3중 방어: (1) RAG 약관 청크 범위 내만 인용, (2) **Z3 SMT 형식 검증**, (3) **이질 LLM cross-check**. 환각율 < 0.5% 목표 (단일 LLM 5~8%). 또한 Advocate 는 차단 장치가 아닌 조언자이며 최종 결정권은 고객에게 있습니다.

#### Q5. "동일 백엔드 LLM 이면 진짜 대립 가능?"
> 본 PoC 는 **Seller=Anthropic Claude, Advocate=OpenAI GPT** 로 **이질 모델 구성**. 서로 다른 학습 데이터·정렬 철학 → 진짜 대립. Phase 1 에서 Llama·Qwen·Solar 추가하여 5종 ensemble 로 확장. MIT MAD 논문의 +80.9% 성능 향상이 **literal 적용**.

#### Q9. "Z3 가 한국어 자연어를 검증한다고?"
> Z3 자체는 자연어를 보지 않습니다. LLM 출력에서 (claim, evidence, source) 트리플을 추출 → 그것을 SMT 변수로 변환 → Z3 가 (1) source 가 KB 내 존재, (2) claim 이 forbidden patterns 부재를 형식 증명. 학술 첫 적용 사례.

#### Q10. "175 에이전트와 Advocate 호환은?"
> A2A 표준으로 통신. Advocate Agent Card 를 `.well-known/agent.json` 에 노출 → 175 에이전트 중 Compliance·Audit 계열이 자동 호출. 표준이라 통합 비용 작음.

#### Q11. "Phase 5 글로벌 진출 — 너무 야심?"
> 야심은 맞지만 근거 있음. Phase 5 까지 누적 470억 / 잠재 수익 1,000~3,000억/년. 우리은행 싱가포르 지점·홍콩 지점 이미 보유. EU 진출은 OCP 인증으로 AI Act 자동 충족 가능.

#### Q12. "행원 입장에서 부담이 크지 않나?"
> 오히려 부담 감소. 행원이 위험 5가지를 외워서 일일이 설명하는 부담을 Advocate 가 떠안음. 행원은 셀링 포인트만 안내 → 면담 시간 단축. Phase 1 시범 영업점 행원 만족도 7.5/10 목표.

#### Q13. "감정 분석으로 고객 음성을 분석한다 — 비윤리적?"
> 영업점 STT 동의를 받은 고객에 한해. 분석은 단말 내 (on-device) — 외부 전송 없음. 결과는 행원 알람용으로만 사용, 저장 X.

#### Q14. "OCP 표준화 — 다른 은행이 따라올까?"
> 3대 압력: (1) 인공지능기본법 고위험 AI 인증의 reference impl, 채택 행정 비용 최소. (2) 금감원 실태평가가 OCP 호환을 2027 권고로 전환 예정 (계획). (3) Linux Foundation 위탁으로 우리은행 종속 제거 — 표준은 중립.

#### Q15. "민원 사전 감지 — Precision/Recall?"
> 베이스라인 측정 후 보정. Phase 1 목표 Precision 70%, Recall 60%. False positive 시 비용은 행원 추가 면담 — 비용 작음. False negative 시 사후 민원 — 비용 큼. 따라서 Recall 우선.

#### Q16. "Advocate 가 부적합 판단을 잘못해서 매출 손해?"
> Phase 1 시범 영업점 측정 KPI: Advocate 인지 후 가입 진행률 67% 목표. 즉 위험 인지 후에도 합리적으로 가입하는 고객이 다수. 가부장 차단 X.

#### Q17. "대안 카테고리 추천 — 책임 회피 vs 진정한 도움?"

#### Q18. "Phase 1 시범 영업점 5개 선정 기준?"

#### Q21. "비용은 누가 대나?"
> P0~P3 누적 270억의 우리은행 자체 부담은 108억 (40%). 나머지는 금융위 R&D·과기정통부·컨소시엄 매칭. DLF급 사태 1회만 회피해도 197억 회수.

> 오히려 반대. 금소법 §17 적합성 + §19 설명의무 강화 트렌드에서 Advocate 는 의무 이행 증거. 모든 비판은 Z3 검증된 약관·통계 인용.

#### Q23. "고객이 Advocate 화면 끄고 가입하면?"

#### Q24. "이거 그냥 가부장적 AI 아닌가?"
> 정반대. 기존 은행 AI 는 '이걸 사세요' 일방향. Advocate 는 '왜 안 맞는지' 항목별로 보여주고 **고객이 직접 체크·서명**. 가부장은 결정 대신해주는 것, Advocate 는 결정을 위한 정보 제공.

#### Q25. "OCP 표준 — 우리은행이 그럴 권한?"
> 권한이 아닌 선점. Linux Foundation A2A 위탁 + 금융위 협력 → 우리은행이 최초 reference impl. KB·신한은 '늦게 따라오는' 포지션.

#### Q26. "그래서 우리은행은 결국 무엇이 되나?"
> **한국 최초의 '금융소비자보호 표준 제정 은행'**. DLF 197억 과태료의 회사가 5년 후 한국 금융권 소비자보호 표준의 출처가 되는 reversal narrative. ESG 1등 + 글로벌 진출 + 정부 협력의 단일 축.

---

### 3-4. 보충 4개

#### Q27. "Phase 0 설계 3개월 — 너무 짧지 않은가?"
> 우리은행 보유 자산이 5개 이미 있어 신규 인프라 거의 0. 설계 3개월은 거버넌스·규제 검토 위주. 실제 코드 빌드는 Phase 1.

#### Q28. "다국어·다문화 고객은?"
> Phase 2 에 다문화 모드 추가. Advocate 시스템 프롬프트만 다국어로 분기. UI 는 i18n. 한국에 거주하는 외국인 노동자·결혼이민 고객 응대.

#### Q29. "본 PoC 에서 실제 작동하지 않는 것은?"
> 정직하게: (1) 이질 LLM 5종 → 현재 2종 (Claude + GPT). (2) Chroma RAG → 현재 정적 청크. (3) 감정 분석 → Phase 2. (4) 고객측 AI 협상 → Agent Card 노출만. (5) OCP 컨소시엄 인증 → PPT narrative. Phase 1 6개월에서 (1)(2)(3) 완성.

#### Q30. "심사 후 즉시 도입 가능한가?"
> Phase 0 설계 3개월 (~2026.09) 후 Phase 1 PoC 6개월 (~2027.03) 에 시범 영업점 5개 가동. 본선 1등이 곧 도입은 아니지만, 우리은행 디지털·AX 본부 직속 과제로 이관 가능.

- proof_artifact.smt2 + JSON trace 첨부

- DLF·홍콩 ELS 급 사태 1건 회피 = 197억 ~ 1.4조 (5개 은행 합산)
- 우리은행 단독 회피 시 197억

**(B) 감경**

- 글로벌 5개국 53개 은행 × $1M + customer fee
- 잠재 1,000~3,000억/년 (Phase 5 완성 시)

### 8-4. Break-even

- Phase 1 (6개월, 20억 누적): DLF 1건 회피로 충당 가능
- 누적 470억은 Phase 5 라이센스 1.6년치로 회수
